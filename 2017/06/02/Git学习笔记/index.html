尽管已经用了一段时间的git，但都属于需要什么查什么，不会什么找什么的状态，这篇博文主要记录我的git学习。

简要介绍

小乌龟SVN是之前用过的一种版本管理系统。但是使用过程中暴露了一些问题，一个是必须在联网状态下使用，另一个是上传速度很慢，10M的东西好几分钟才能传完。

在这里先简单说一下两类版本控制系统：

  • 集中式版本控制系统
  • 分布式管理系统

集中式版本控制系统，如CVS与SVN，他的工作模式类似与去图书馆借书，所有的资源集中存放在中央服务器上。
Repository（版本库），的每次更新都需要联网获取上一版本，然后修改，再上传到中央服务器上。
分布式版本控制系统，目前Git是这一类的佼佼者，也是github唯一指定的版本工具。他没有中央服务器，版本库可以在自己的电脑上离线使用，也可以在联网情况下与他人共享。另一方面也反应了他的安全性，不用担心在传输过程中掉入黑洞。

安装设置

git安装很简单，在git官网上有一个大大的Download。重点是一些安装之后的设置。
$ git config –global user.name “OrangeBorning”
$ git config –global user.email “orangeborningzxp@gmail.com”
这里主要设置这台电脑内所有git仓库的全局用户名和邮箱，留下犯罪的罪证。

通过以上操作，就完成了简单的git环境设置，暂别git，下面进行github上的操作。

github作为最受欢迎的代码托管平台，也只支持git的作为他的版本管理系统。github的注册其实很简单：

右上角点击sign up

  • step1 输入Username，password，email(注册的邮箱很重要)
  • step2 直接continue
  • step3 点击skip

只需三步完成注册。接下来要配置你的SSH。
注册完成登录后，右上角点击你的头像，打开setting，找到SSH and GPG keys，里面有完整的英文指南，其实只需要三步：

  • step1 generating 命令行中输入以下内容生成你的密钥

    $ ssh-keygen -t rsa -C “your_email@example.com你注册时的邮箱”

  • step2 adding 命令行中输入一下内容复制你的密钥，打开SSH and GPG keys中的New SSH key，右键粘贴，title随便起啦

    $ clip < ~/.ssh/id_rsa.pub

  • step3 testing 完成以上两部你应该会在github中看见你的黑钥匙，接下来输入以下内容让他变绿。

    $ ssh -T git@github.com

完成以上操作你的git与github算是连接成功了。下面用一个小李子说明下git仓库的操作。

git仓库

首先我们创建一个文件夹,名字随便起，例如foldername

$ mkdir foldername

然后进入这个这文件夹

$ cd foldername

在这个项目（就是这个文件夹，之后我们叫他项目）中打开命令行

$ git init

生成我们的本地仓库，这是一个empty Git repository，在这部操作后项目中会生成一个隐藏文件夹.git。在这里面有我们的储存区和缓存区（index）。没有必要的话不用修改里面的内容。

下面我们touch一个.gitignore文件（突然想起高中和我们菲律宾外教争辩ignore应该怎么读的场景）。

$ touch .gitignore

这时候你的项目中应该会有一个.gitignore文件，这个文件可以添加你希望git仓库忽略的文件或文件夹，填入相应的名字即可。

接下来我们在项目中添加一个任务

$ touch test.md

项目中生成一个名为test.md的文件，进入这个文件添加一些内容。

$ git status

我们先看看仓库目前的状态，发现有两个红名文件，我在执行下一步。

$ git add .

注意这里add空格后是一个 .

$ git status

再次执行后，我们发现名字变绿了，就和钥匙变绿一样，什么都变绿了就好了。

$ git commit -m 添加了一个test.md文件

-m需要你填写这次提交仓库的信息，以便别人看懂你做了什么。

到这里你已经完成了本地仓库的创建，我们需要把他连接到我们的远程仓库github上，接下来进入github页面。

右上角头像旁边的加号点击下Create a new repository，这里只需要写上你这个仓库的名字，和对仓库的描述，描述随时都可以改。

完成后你就进入了你的repository，点击ssh复制这个地址，我们再回到git bash中

$ git remote add origin git@github.com:yourusername/repositoryname.git

这部操作是连接你的本地仓库与远程仓库,成功后我们执行

$ git push -u origin master

好了，我们现在刷新github的项目，发现他成功的添加了我们本地仓库的内容。
到这里一个本地仓库与远程仓库的连接就算完成了。之后我们每次修改了什么内容执行这几部操作就可以

$ git pull
$ git add .
$ git commit -m 修改信息
$ git push

简单上传步骤就是如此。

下面说说如何让远程仓库的内容在我的另一台机器上clone一个本地仓库。还记得你的远程仓库地址吗，复制他。

$ git clone git@github.com:yourusername/repositoryname.git

执行这部操作后，你的文件夹中会得到一个来自github的clone项目。
之后的操作和最开始的本地操作一样

$ git pull
$ git add .
$ git commit -m 修改信息
$ git push

这几个指令不是一起输入，我是说最简单的就需要这几个。git更深的学习慢慢会更新，最后如果想继续深造的可以试试这个游戏
